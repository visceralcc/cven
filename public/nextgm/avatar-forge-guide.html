Overview

What It Is

Avatar Forge is a private web tool that generates illustrated player portraits for NextGM using AI image generation via fal.ai. It lives at cven.cc/nextgm/avatar-forge and runs on Cloudflare Workers — no server to manage, no login required.

The tool lets you choose a model, set filters for player attributes, and generate batches of portraits in one click. The goal is to eventually produce all 2,700 avatars needed for the full NextGM player pool.

02

Generation Settings

Batch Size

How many images to generate in one run. Start with 3–5 when testing a new prompt or model. Scale up once you're happy with the output.

Concurrency

How many images are generated simultaneously. At 3, it fires 3 requests at once, waits for each to finish, then starts the next group. Higher = faster batches, but you can hit fal.ai rate limits. The default of 3 is safe to leave as-is.

Image Size

Square HD (512×512) is the standard for player avatars. Portrait 3:4 gives more vertical room if you want to show more of the chest and shoulders.

03

Variation Matrix (Filters)

The filters control what gets randomly selected for each image in a batch. All filters start off (gray) — click to toggle on (green). At least one option must be active in each category before you can generate.

lean athletic stocky beefy round

When you run a batch, each image independently picks one random value from each active filter and plugs it into the prompt. So if you have Black and White both active, the batch will include a mix of both.

To generate one specific type — say, only Black players with buzz cuts — activate only those specific tags and leave everything else off.

Filter Categories

CategoryWhat it controls
EthnicityThe player's described racial/ethnic background. Also determines which style reference image is used (Kontext only).
HairstyleHair description passed to the model — buzz cut, dreads, afro, fade, man bun, etc.
Body TypePhysical build — lean, athletic, stocky, heavy-set, beefy, round.
Facial HairClean-shaven, stubble, full beard, goatee, mustache, salt & pepper.
AccessoriesEarring styles — none, studs, hoops, diamonds, single hoop.
04

Style Board & Reference Images

The Style Board shows the reference images currently loaded for each ethnicity. These are used exclusively with Kontext Pro — one is randomly picked per image to guide the illustration style.

  • Each ethnicity has its own folder of reference images
  • When Kontext Pro runs, it picks one ref from the matching ethnicity folder
  • If an ethnicity has no refs, no reference is sent — it falls back to pure text-to-image
  • Inactive ethnicity tags (toggled off) appear dimmed in the style board
  • Each bucket shows a green ref count or a red "no refs" warning

Adding New Reference Images

This is a two-step process. Files must be added locally AND the code must be synced before deploying.

Step 1: Drop your new PNG files into the correct ethnicity folder on your Mac:

navigate to the refs folder for the ethnicity you want ~/dev/cven/public/nextgm/refs/black/ ~/dev/cven/public/nextgm/refs/white/ ~/dev/cven/public/nextgm/refs/hispanic/ ~/dev/cven/public/nextgm/refs/polynesian/ ~/dev/cven/public/nextgm/refs/asian/ ~/dev/cven/public/nextgm/refs/mixed/

Step 2: Run the sync script and deploy:

cd ~/dev/cven node sync-refs.js npx wrangler deploy
The sync-refs.js script auto-scans all ref folders and updates the image list in the HTML. You never need to edit that list manually — just run the script after adding files, then deploy.

Naming Convention

Files can be named anything. The existing convention is NN_ethnicity_descriptor.png — for example, 01_black_buzzcut.png or 06_white_blonde.png. The number is just for sorting. Hover a thumbnail in the style board to see the filename.

The Prompt

The text box in the Style Template section is the actual instruction sent to the AI model for every image in the batch. Variables in curly braces get replaced with random values from your active filters on each generation.

Variables

VariableReplaced with
{ethnicity}Random active ethnicity tag (e.g. "Black", "Hispanic Latino")
{age}Random age between 21–38
{hairstyle}Random active hairstyle tag
{body_type}Random active body type tag
{build}Random neck/shoulder description (hardcoded — e.g. "muscular neck and broad shoulders")
{facial_hair}Random active facial hair tag
{accessories}Random active accessories tag
{expression}Random expression from a hardcoded list (confident, serious, stoic, etc.)
{skin_tone}Random skin tone matched to the selected ethnicity

Editing the Prompt

You can edit the prompt text directly in the browser. Changes take effect immediately — no deploy needed. The prompt resets to the baked-in default any time you hard-refresh the page (Cmd+R).

If you find a prompt version that works well and want it to become the permanent default, ask Claude to update the HTML and deploy it.
06

Deploying Changes

All changes to the tool — UI updates, new ref images, baked-in prompt changes — require a deploy to go live. The tool is hosted on Cloudflare Workers.

Always deploy from the project root:

cd ~/dev/cven npx wrangler deploy

Deploy takes about 10 seconds. Changes are live immediately after — no cache to clear.

What Requires a Deploy

  • Adding new reference images (after running sync-refs.js)
  • Changing the default prompt text permanently
  • Adding new filter tags or values
  • Any UI or layout changes

What Does NOT Require a Deploy

  • Editing the prompt in the browser (session-only, resets on refresh)
  • Toggling filter tags on/off
  • Changing model, batch size, or concurrency

Troubleshooting

I added images to a refs/ folder but they don't appear in the style board

You need to run node sync-refs.js and then npx wrangler deploy. Adding files to the folder alone is not enough — the HTML has a list that must be updated by the script first.

The prompt isn't saving between sessions

The prompt only persists until you refresh the page. If you want a custom prompt to survive refreshes, ask Claude to bake it into the HTML and deploy.

Generation failed or timed out

fal.ai requests occasionally time out, especially under load. The tool logs errors in the Generation Log at the bottom of the main panel. A few failures in a large batch are normal — re-run a smaller batch to fill in the gaps.

Output images look too much like the reference image

This is a Kontext Pro behavior — it weights the style reference heavily. The best fix is to build up more refs per ethnicity so the model has more variety to draw from, rather than anchoring on a single image each time.

Outputs don't match the selected ethnicity

Make sure only the ethnicity tags you want are toggled on (green). If multiple are active, the batch will be a mix. Also confirm the refs folder for that ethnicity has images — check the style board for the green ref count badge.

The style board shows "no refs" for an ethnicity

That ethnicity folder is empty. If you generate with Kontext Pro and that ethnicity is selected, it'll fall back to pure text-to-image (no style reference). Add ref images to that folder, run sync-refs.js, and deploy to fix it.
